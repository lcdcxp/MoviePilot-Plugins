# 监控strm刮削网盘

复用 MoviePilot 全局媒体库入库事件，判断 STRM 库里的刮削信息是否完整；需要刮削时，再把 STRM 路径映射到网盘真实路径并触发 MP 刮削。

## 推荐配置

- STRM 检查根路径：`/media`
- MP 刮削目标根路径：`/CD2/115/CMS影库/影视`
- 入库后检查延迟秒：`10`
- 单集刮削等待分钟：`6`
- 电视剧复查天数：`10`

路径示例：

`/media/电影/华语电影/片名/xxx.strm`

检查目录：`/media/电影/华语电影/片名`

刮削目标：`/CD2/115/CMS影库/影视/电影/华语电影/片名`

## 逻辑

电影：检查片名目录是否同时存在 `backdrop.jpg`、`fanart.jpg`、`poster.jpg` 和任意 `.nfo`。完整跳过，不完整立即刮削一次。

电视剧/番剧：检查 STRM 所在剧名根目录。根目录没有任何图片/nfo 时刮削整部剧；已有根目录信息时，只检查本次入库单集是否有对应图片，缺图则删除同名 `.nfo`，6 分钟后刮削该集。入库 10 天后复查全剧所有 STRM，缺图单集逐集处理。

媒体库过滤会同时识别路径第一层和第二层，例如 `/media/电视剧/国产剧/...` 可以命中 `电视剧` 或 `国产剧`。

历史记录最多保留最新 100 条，可在配置页清空；队列操作不会删除实际文件。
