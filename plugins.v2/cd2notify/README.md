# CloudDrive2通知（CD2Notify）

作者：jidian  
插件 ID：`CD2Notify`  
当前版本：`v1.2`

CD2Notify 用来接收 **CloudDrive2 Webhook**，把文件新增、删除、移动、重命名和挂载状态整理成 **MoviePilot 通知**。批量操作时可开启汇总通知，等连续变化结束后再合并发送一条消息。

```text
CloudDrive2 文件变化 / 挂载变化
        ↓
CD2Notify 校验 apikey 并解析事件
        ↓
MoviePilot 通知渠道发送
```

## 支持的动作

| CloudDrive2 动作 | 通知显示 |
|---|---|
| `create` | 新增 |
| `delete` | 删除 |
| `rename` | 按路径判断为移动、重命名或移动并重命名 |
| `mount` / `unmount` | 挂载 / 卸载 |

说明：CloudDrive2 没有单独的 `copy` 动作，复制文件通常会被识别为 **新增**。

## 安装方法

1. 在 MoviePilot 安装 **本地插件安装** 插件。
2. 上传本插件 ZIP 包。
3. 在插件列表里找到 **CloudDrive2通知**。
4. 打开插件并启用。

## 推荐设置

| 设置项 | 建议 |
|---|---|
| 启用插件 | 开 |
| 开启通知 | 开 |
| 消息类型 | Manual / 手动处理 |
| 显示具体文件名称 | 开启后通知显示详情和完整路径；关闭后 MP 通知隐藏详情，历史记录和插件日志仍保留完整路径 |
| 最多显示详情条数 | 默认 10；填 0 表示不限制 |
| 历史记录保留条数 | 默认 50；填 0 表示不记录 |
| 启用汇总通知 | 批量复制、删除、移动时建议开启 |
| 汇总等待时间（秒） | 默认 20 |

如果使用 **企微机器人消息推送**，也要确认企微插件允许发送当前消息类型。

## CloudDrive2 Webhook 配置

插件包里自带模板：

```text
cd2_webhook_template.toml
```

只需要改两处：

```text
base_url = "http://你的MP地址:端口"
apikey = "你的MP_API_TOKEN"
```

文件变动通知地址：

```text
你的MP地址/api/v1/plugin/CD2Notify/file_notify?apikey=你的MP_API_TOKEN
```

挂载状态通知地址：

```text
你的MP地址/api/v1/plugin/CD2Notify/mount_notify?apikey=你的MP_API_TOKEN
```

注意：这两个地址是填到 **CloudDrive2 Webhook 配置** 里的，不是填到 MP 通知渠道里。公网反代时建议使用 HTTPS，并尽量限制来源 IP；不要公开真实 `apikey`。

## 汇总通知

开启 **启用汇总通知** 后，插件会先缓存连续的文件变化：

```text
收到文件变化
  ↓
等待 N 秒
  ↓
N 秒内又收到新变化则重新计时
  ↓
直到 N 秒内没有新变化，再统一发送一条汇总通知
```

说明：这个功能不是直接读取 CloudDrive2 的任务队列，而是通过“多久没有新的 webhook”判断一批操作基本结束。

## 查看数据

点击插件的 **查看数据**，可以看到最近的 CloudDrive2 操作历史，包括：

```text
时间
类型
操作
数量
说明
```

即使关闭 **显示具体文件名称**，查看数据里的历史记录也会显示具体路径，方便排查是哪一个文件或目录触发通知。

## 测试方法

保存 CloudDrive2 Webhook 配置后，建议重启 CloudDrive2。然后依次测试：

1. 新建一个文件夹，看是否通知“新增目录”。
2. 删除这个文件夹，看是否通知“删除目录”。
3. 重命名一个文件夹，看是否通知“重命名目录”。
4. 移动一个文件夹，看是否通知“移动目录”。
5. 在插件配置里打开“测试通知”并保存，看通知渠道是否能收到测试消息。

## 常见问题

### 没收到通知怎么办？

检查：

1. CD2Notify 插件是否启用。
2. 插件里的“开启通知”是否打开。
3. CloudDrive2 模板里的 MP 地址是否正确。
4. URL 里的 `apikey` 是否等于 MoviePilot 系统 `API_TOKEN`。
5. 通知渠道是否允许当前消息类型。
6. 如果开启汇总通知，等待时间内不会立即发送，需要等连续变化停止后再看。

### 为什么没有复制通知？

CloudDrive2 没有 `copy` 动作，复制文件通常显示为 **新增**。

### 关闭“显示具体文件名称”后还能排查吗？

可以。关闭后只隐藏 MP 通知里的详情，插件日志和“查看数据”历史记录仍保留完整路径。

## v1.2 更新说明

- 统一代码、manifest 和文档版本号。
- 修复汇总通知在配置读取时默认值不一致的问题。
- Webhook `apikey` 使用常量时间比较，降低 token 比较侧信道风险。
- 对异常大的 Webhook 回调做条数和字段长度限制，避免通知、日志和历史记录被撑爆。
- 修复保存配置时待汇总事件被静默清空的风险。
- 测试通知和正式通知都会识别通知通道显式返回失败的情况。
