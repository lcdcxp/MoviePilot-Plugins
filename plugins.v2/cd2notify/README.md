# CloudDrive2通知（CD2Notify）

作者：jidian  
插件 ID：`CD2Notify`

这个插件用来接收 **CloudDrive2 Webhook**，把文件新增、删除、移动/重命名整理成更清楚的 **MoviePilot 通知**。也可以把连续变化合并，等一批操作结束后再发一条汇总通知。

```text
CloudDrive2 文件变化
        ↓
CD2Notify 解析动作和路径
        ↓
MoviePilot 通知渠道发送
```

## 支持的动作

| CloudDrive2 动作 | 通知显示 |
|---|---|
| `create` | 新增 |
| `delete` | 删除 |
| `rename` | 移动/重命名 |

说明：CloudDrive2 没有单独的 `copy` 动作，复制文件通常会被识别为 **新增**。

## 通知效果

新版通知更简洁，带分隔线和图标：

```text
【☁️CloudDrive2】新增目录

━━━━━━━━━━━━━━
📊 本次变动：1 项
➕ 新增：1 项｜🗑️ 删除：0 项｜🔁 移动：0 项
📄 文件：0 项｜📁 目录：1 项
📌 详情
➕ 新增目录：/115/nas/111
━━━━━━━━━━━━━━
🕐 时间：2026-06-01 09:52:39
```

如果关闭 **显示具体文件名称**，MP 通知不会展示具体路径；点击 **查看数据** 时，历史记录仍会显示完整路径，方便排查。

## 安装方法

1. 在 MoviePilot 安装 **本地插件安装** 插件。
2. 上传本插件 ZIP 包。
3. 在插件列表里找到 **CloudDrive2通知**。
4. 打开插件并启用。

## 插件推荐设置

| 设置项 | 建议 |
|---|---|
| 启用插件 | 开 |
| 开启通知 | 开 |
| 消息类型 | Manual / 手动处理 |
| 显示具体文件名称 | 开启后通知显示详情和完整路径；关闭后MP通知不显示详情，历史记录和插件日志仍保留完整路径 |
| 最多显示详情条数 | 10，填0表示不限制 |
| 历史记录保留条数 | 50 |
| 启用汇总通知 | 按需开启，连续变化合并成一条通知 |
| 汇总等待时间（秒） | 建议 20 |

如果使用 **企微机器人消息推送**，也要确认企微插件允许发送 `Manual / 手动处理` 类型通知。

## 启用汇总通知

如果你一次性复制、删除、移动很多文件，CloudDrive2 可能会连续发送很多次 webhook。

开启 **启用汇总通知** 后，插件不会马上每条都推送，而是先缓存起来：

```text
收到文件变化
  ↓
等待 N 秒
  ↓
如果 N 秒内又收到新变化，就重新计时
  ↓
直到 N 秒内没有新变化，再统一发一条汇总通知
```

推荐设置：

```text
启用汇总通知：开
汇总等待时间（秒）：20
```

说明：这个功能不是直接读取 CloudDrive2 的任务队列，而是通过“多久没有新的 webhook”来判断一批操作基本结束。


## 显示具体文件名称是什么意思？

- 统计信息默认开启，通知里会显示本次变动、新增、删除、移动、文件和目录数量。
- **显示具体文件名称**：控制 MP 通知里是否显示“📌 详情”和完整路径。开启后通知显示详情和路径；关闭后 MP 通知不显示详情，但“查看数据”和插件日志仍会保留完整路径，方便自己排查。

简单理解：

```text
统计信息 = 默认显示数量统计
显示具体文件名称 = 是否在 MP 通知里显示详情和完整路径
启用汇总通知 = 是否等待一批操作结束后再统一发一条
```

## CloudDrive2 Webhook 怎么填

插件包里自带完整模板：

```text
cd2_webhook_template.toml
```

只需要改两处：

```text
base_url = "http://你的MP地址:端口"
apikey=你的MP_API_TOKEN
```

文件变动通知地址：

```text
你的MP地址/api/v1/plugin/CD2Notify/file_notify?apikey=你的MP_API_TOKEN
```

挂载状态通知地址：

```text
你的MP地址/api/v1/plugin/CD2Notify/mount_notify?apikey=你的MP_API_TOKEN
```

注意：这两个地址是填到 **CloudDrive2 Webhook 配置** 里的，不是填到 MP 通知渠道里。

## 查看数据

点击插件的 **查看数据**，可以看到最近的 CloudDrive2 操作历史，包括：

```text
时间
类型
操作
数量
说明
```

即使关闭 **显示具体文件名称**，查看数据里的历史记录也会显示具体路径，方便排查是哪一个文件或目录触发通知。

## 测试方法

保存 CloudDrive2 Webhook 配置后，建议重启 CloudDrive2。

然后依次测试：

1. 新建一个文件夹，看是否通知“新增目录”。
2. 删除这个文件夹，看是否通知“删除目录”。
3. 重命名一个文件夹，看是否通知“移动/重命名目录”。

## 常见问题

### 没收到通知怎么办？

检查：

1. CD2Notify 插件是否启用。
2. 插件里的“开启通知”是否打开。
3. CloudDrive2 模板里的 MP 地址是否正确。
4. URL 里的 `apikey` 是否等于 MoviePilot 系统 API_TOKEN。
5. 企微机器人插件是否允许当前消息类型。

### 为什么没有复制通知？

CloudDrive2 没有 `copy` 动作，复制文件通常显示为 **新增**。
